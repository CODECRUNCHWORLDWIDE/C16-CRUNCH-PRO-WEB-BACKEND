## Tools to install this week

- **`django-extensions`** — adds `shell_plus`, `runserver_plus`, and the brilliant `show_urls` command:
  ```bash
  pip install django-extensions
  ```
  Add `'django_extensions'` to `INSTALLED_APPS`.

- **`django-debug-toolbar`** — see every query Django runs in your page response:
  <https://django-debug-toolbar.readthedocs.io/>
  ```bash
  pip install django-debug-toolbar
  ```

- **`ipython`** — a much better Python REPL. With `django-extensions`, `python manage.py shell_plus` will auto-import all your models and use IPython.

## Glossary

| Term | Definition |
|------|------------|
| **Model** | A Python class that maps to a database table |
| **Field** | A model attribute that maps to a column |
| **Migration** | A versioned schema-change script generated by Django |
| **QuerySet** | A lazy, chainable representation of a database query |
| **N+1 problem** | Fetching a parent + N children with N+1 separate SQL queries instead of 1 or 2 |
| **`select_related`** | Eager-loads a foreign-key relationship using a SQL JOIN (one query) |
| **`prefetch_related`** | Eager-loads a many-to-many or reverse FK with a second query + Python join |
| **ModelAdmin** | The class that controls how a model appears in the admin |
| **Superuser** | A Django user with all permissions; created via `createsuperuser` |
| **`on_delete`** | What happens to a row when the referenced row is deleted (CASCADE, PROTECT, SET_NULL, etc.) |
| **`related_name`** | The name of the reverse accessor on the related model |
| **`Meta` class** | Inner class on a model that sets options not tied to a single field |
